- 2026-03-26: Area monitor drawing now sets its own global drawing-mode flag and temporarily removes parcel click handlers while active. Rationale: parcel layers were intercepting clicks that needed to reach the map, so matching the existing road/track interaction model was the least risky fix.
- 2026-03-26: Area monitor rendering now separates “reapply styles” from “fly to monitor”, and monitor open/close flows go through explicit routing helpers plus a list endpoint/modal. Rationale: parcel refreshes were causing unintended camera jumps and monitor cleanup needed one reliable path that resets overlay and parcel highlighting together.
- 2026-03-27: The Docker dev backend now runs `npm install` before `nodemon` on container start. Rationale: the bind-mounted `/app` plus separate `/app/node_modules` volume was leaving containers with stale dependencies after backend package changes.
- 2026-03-27: The backend now boots through `backend/server.js`, and production PM2 runs it in `fork` mode instead of `cluster` mode. Rationale: the import-safe app module and PM2 cluster setup left the API process online without binding port 3000, while the explicit server entrypoint plus fork mode restored reliable startup.
- 2026-03-27: The backend deploy script now uses `pm2 startOrRestart ecosystem.config.cjs --update-env`. Rationale: plain stop/start was preserving stale PM2 app definitions instead of reliably applying ecosystem changes like `fork` mode and new script paths.
- 2026-03-28: Zagreb parcel bbox reads keep `parcel_info` ownership, but now fetch the latest details row per parcel key with lateral lookups instead of precomputing a whole-table latest-version CTE. Rationale: the whole-table `parcel_info` join timed out in production under map tile fan-out, while key-based lookups preserved the richer ownership source without reverting to `parcel_detail`.
- 2026-03-28: Backend deploys now run the public `/health` plus `/parcels?bbox=...` smoke test after PM2 restart, with bounded retries before the deploy is considered failed. Rationale: the recent production outage was a post-restart/runtime regression that ordinary file sync plus PM2 success did not catch.
- 2026-03-28: The area monitor detail panel now supports a header-only minimized state and dynamically stacks above the parcel info panel when that panel is open. Rationale: on mobile, closing the monitor panel discarded the monitor context, while leaving it full-height blocked too much of the map and collided with parcel info.
- 2026-03-31: Area monitors now persist a canonical `city_id`, legacy monitors infer Zagreb on read, and monitor routing blocks cross-city loads until the user switches cities or cancels. Rationale: shared monitor URLs were loading the wrong parcel datasource after city changes, so storing and enforcing monitor city was the cleanest fix.
- 2026-04-14: Synthetic descendant parcel IDs must stay rooted at the base parcel ID, and road replay only skips a remainder when the area delta is within a tiny absolute tolerance. Rationale: shared-plan replays were regenerating nested synthetic IDs and dropping valid small remainders, which broke old proposal dependency chains.
- 2026-04-14: Frontend deploys now stamp the build token into `index.html` and version every local script URL from the HTML bootstrap. Rationale: Cloudflare was caching plain `js/*.js` paths after deploy, so the live app kept booting stale runtime code even though the origin files were updated.
- 2026-04-14: Shared-plan replay now treats server-provided child parcel IDs as canonical for road and reparcellization proposals, and synthetic ID composition strips inherited synthetic suffixes defensively. Rationale: reusing locally-regenerated child IDs caused stale nested descendants and missing downstream prerequisites even when the shared payload already had the correct canonical IDs.
- 2026-04-14: The `/planned-road` route now clips `planned_road.geom` to the request bbox before `ST_Difference`, and builds props from explicit non-geometry columns instead of `to_jsonb(pr) - 'geom'`. Rationale: the old query was serializing a citywide geometry for small viewports and, after `ST_Dump`, repeatedly re-encoding the discarded source geometry into props, turning the government road plan toggle into a multi-second stall.
- 2026-04-14: Loading the government road plan overlay now defers plan hashing and full Turf union until the apply path actually needs it. Rationale: eager remaining-plan initialization made the simple “show plan” checkbox do heavy main-thread geometry work at zoomed-out views, freezing the page even though the user only asked to draw the overlay.
- 2026-05-11: Solana proposal authorization now validates parcel ownership against the fixed ParcelNFT program ID instead of storing a parcel program on each proposal. Rationale: this closes spoofed-acceptance risk without changing the serialized proposal account layout.

- 2026-07-08: Building footprints (2D + measured heights) are an OPTIONAL per-city provider capability: a provider in `backend/buildings/` that implements `footprints(geometry)` is served by `POST /buildings/footprints`; anything else reports `supported: false` (route resolves cities exactly, no Zagreb fallback). Zagreb implements it from `building` JOIN `building_3d_match.height_m`. Rationale: powers the urban-rule "Based on existing buildings" mode (two exclusive radio-picked rules — extrude every footprint to an exact proposed height, or add N floors on top of each current height, with an adjustable floor-height slider 2.5–5 m; one Feature per building with its own `properties.height`) and makes new cities work the moment their provider adds the method. Known caveat: Zagreb cadastre footprints don't truly match the `building_3d` meshes — at some point we may need to reverse-derive footprints from the 3D data.
