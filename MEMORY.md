# Memory

- 2026-03-26: Area monitor drawing now sets its own global drawing-mode flag and temporarily removes parcel click handlers while active. Rationale: parcel layers were intercepting clicks that needed to reach the map, so matching the existing road/track interaction model was the least risky fix.
- 2026-03-26: Area monitor rendering now separates “reapply styles” from “fly to monitor”, and monitor open/close flows go through explicit routing helpers plus a list endpoint/modal. Rationale: parcel refreshes were causing unintended camera jumps and monitor cleanup needed one reliable path that resets overlay and parcel highlighting together.
- 2026-03-27: The Docker dev backend now runs `npm install` before `nodemon` on container start. Rationale: the bind-mounted `/app` plus separate `/app/node_modules` volume was leaving containers with stale dependencies after backend package changes.
- 2026-03-27: The backend now boots through `backend/server.js`, and production PM2 runs it in `fork` mode instead of `cluster` mode. Rationale: the import-safe app module and PM2 cluster setup left the API process online without binding port 3000, while the explicit server entrypoint plus fork mode restored reliable startup.
- 2026-03-27: The backend deploy script now uses `pm2 startOrRestart ecosystem.config.cjs --update-env`. Rationale: plain stop/start was preserving stale PM2 app definitions instead of reliably applying ecosystem changes like `fork` mode and new script paths.
