In Parcel Info proposals tab, in each proposal list item we need to add a button called Compare. The Compare button launches a modal which has a table with two columns for each of the rows (row labels) below. The first column is titled Currently, the second Proposed. The rows are: 1) parcel area, 2) building footprint, 3) building height, 4) building floors, 5) square meters, 6) parking spots. 7) estimated market value of the building 
Difference in market value (potential profit): current makret value subtracted from new market value

How to calculate the values:
1) parcel area
current: area of the parcel
new: area of the parcel (should be same for building proposals and different if the proposal is a road splitting or changing the parcel)

2) building footprint
current: the area of the building(s) on the parcel currently
new: area of the part of the block building polygon that intersects the parcel

3) building height
current: put current height, which we assume to be 10 meters
new: depends on the height of the proposed building, pull from there

4) building floors
current: current height / 3, round down
new: proposed height / 3, round down

5) square meters
current: current footprint x current floors
new: new footprint x new floors

6) parking spots
current: 4
new: 0

7) estimated market value of the building
current: current square meters x square meter price in the area (assume 3500 EUR for now)
new: new square meters x square meter price in the area (assume 3500 EUR for now)
