Difference in market value (potential profit): current market value subtracted from new market value. How to calculate the values:
  1) parcel area
  current: area of the parcel
  new: area of the parcel (should be same for building proposals and different if the proposal is a road splitting or changing the parcel)
  2) building footprint
  current: the area of the building(s) on the parcel currently
  new: area of the part of the block building polygon that intersects the parcel
  3) building height
  current: put current height, which we assume to be 10 meters
  new: depends on the height of the proposed building, pull from there
  4) building floors
  current: current height / 3, round down
  new: proposed height / 3, round down
  5) square meters
  current: current footprint x current floors
  new: new footprint x new floors
  6) parking spots
  current: 4
  new: 0
  7) estimated market value of the building
  current: current square meters x square meter price in the area (assume 3500 EUR for now)
  new: new square meters x square meter price in the area (assume 3500 EUR for now)

- First we need to solve the situation where there are multiple proposals for the same parcel or same block. We need to make it so only one proposal can be displayed on the map at one time.

- Secondly, there is the fact that proposals can depend on other proposals. For example, a new road is drawn, it forms two blocks of parcels out of one, and on one of the resulting blocks a building is proposed. So the building proposal depends on the road proposal (being accepted and executed first).

- I am thinking the best way to track this is by having each object have a parent_proposal pointer, and each proposal having a list of "parent parcels", that is, parcels affected by the proposal. Initially, all parcels and buildings should have the parent(s) empty or null, because when we load them they represent the current state of the world. But when a new proposal is added, it becomes the parent for the changes it introduced. Example: if we create a road proposal for a road that split a parcel in two, it would have one parent (the old parcel) and three children: the road parcel itself and the two new, smaller parcels that inherited the old parcel.

1) select from multiple proposals (to display a proposal we need to check if all its ancestors are visible on the map unaffected)
